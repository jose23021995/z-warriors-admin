Análisis completo del componente CharacterListComponent:

## 1. **Archivos del componente:**
   - **character-list.component.ts**: Componente principal en TypeScript.
   - **character-list.component.html**: Plantilla HTML.
   - **character-list.component.scss**: Estilos CSS/SCSS.
   - **character-list.component.spec.ts**: Archivo de pruebas unitarias.

## 2. **Funcionalidad general:**
   El componente CharacterListComponent es la vista principal de personajes en el dashboard. Muestra una lista paginada y filtrable de personajes de Dragon Ball, permitiendo ver detalles, editar y gestionar la información. Actúa como contenedor principal que integra una tabla de personajes con funcionalidades de carga, búsqueda y modales para detalles y edición.

## 3. **Dónde es invocado:**
   - **Rutas**: Definido en `app.routes.ts` como ruta hija de `/dashboard/characters`.
   - **Lazy Loading**: Se carga de forma diferida con `loadComponent`.
   - **Acceso**: Usuario accede vía menú "Personajes" en el dashboard, o por defecto al entrar a `/dashboard` (redirect automático).
   - **Ubicación en jerarquía**: `/dashboard/characters` → CharacterListComponent.

## 4. **Uso de servicios:**
   - **CharacterService**: Principal servicio utilizado.
     - `getCharacters(page, limit, search)`: Obtiene lista paginada de personajes desde la API externa.
     - `getCharacter(id)`: Obtiene detalles de un personaje específico para modales.
   - Los datos se cargan de forma asíncrona usando `firstValueFrom` implícitamente en el servicio.

## 5. **Procesamiento y manejo de datos:**
   - **Carga inicial**: En `ngOnInit()`, llama a `handleLoadData({ page: 1, limit: 10 })`.
   - **Paginación y búsqueda**: `handleLoadData()` recibe eventos de la tabla (página, límite, búsqueda).
     - Sin búsqueda: Carga datos normales con paginación.
     - Con búsqueda: Filtra localmente por nombre o raza (case-insensitive).
   - **Estados**: Usa signals para `characters` (array de personajes), `totalCount` (total de registros), `isLoading` (estado de carga).

## 6. **Interfaz de usuario y componentes hijos:**
   - **HeadersComponent**: Muestra el título "PERSONAJES".
   - **CharacterTable**: Componente hijo principal que muestra la tabla.
     - Props: `[characters]`, `[totalRecords]`, `[loading]`.
     - Eventos: `(onLoadData)` para paginación/búsqueda, `(onCharacterSelected)` para selección de personaje.
     - El componente no hace llamadas HTTP propias; simplemente renderiza datos y emite eventos al padre.
     - Usa `triggerSearch()` para buscar desde el input de texto y `loadCharactersLazy()` para cambiar página de manera lazy.
   - **Modales**:
     - **CharacterDetailComponent**: Abre para ver detalles (solo lectura).
     - **CharacterForm**: Abre para editar personaje.
   - **Spinner**: Muestra `p-progressSpinner` mientras carga.

## 7. **Manejo de eventos y modales:**
   - **abrirEdicion(guerrero)**: Abre modal de detalles (parece obsoleto, ya que `handleCharacterUpdate` lo reemplaza).
   - **handleCharacterUpdate(data)**: Maneja selección de personaje de la tabla.
     - Extrae `{character, type}` del evento.
     - Obtiene datos completos con `getCharacter(character.id)`.
     - Si `type` es `false`: Abre `CharacterDetailComponent` en modal con header "Informacion de {name}" y width 60%, pasando `{response}`.
     - Si `type` es `true`: Abre `CharacterForm` en modal con header "Editar a {name}" y width 90%, pasando `{response}`.
     - Los modales se abren con `DialogService.open()`, y se puede suscribir a `onClose` para recibir datos de retorno.
   - **DialogService**: Gestiona apertura y cierre de modales dinámicos.

## 7.1 **Flujo de datos entre padre e hijo:**
   - `CharacterListComponent` es el componente padre que obtiene y prepara los datos.
   - El hijo `CharacterTable` recibe tres props del padre: `characters`, `totalRecords` y `loading`.
   - `CharacterTable` muestra estos datos en la tabla y no realiza llamadas HTTP por sí mismo.
   - Cuando el usuario busca o cambia de página, `CharacterTable` emite `onLoadData` con `{ page, limit, search }`.
   - El padre `CharacterListComponent` recibe ese evento y usa `CharacterService.getCharacters()` para cargar datos nuevos desde la API.
   - Cuando el usuario selecciona una acción en la tabla, `CharacterTable` emite `onCharacterSelected` con `{ character, type }`.
   - El padre recibe el evento y decide si abre `CharacterDetailComponent` o `CharacterForm`, completando el ciclo hijo → padre.

## 7.2 **Cómo se usan los datos entre componentes y servicios:**
   - El padre pide datos al servicio `CharacterService` en `handleLoadData()`.
   - El servicio devuelve `response`, que puede contener `items` o ser directamente un array.
   - El padre normaliza esa respuesta en `rawData` y actualiza los signals `characters` y `totalCount`.
   - Esos signals se inyectan en `CharacterTable`, donde se renderizan como `displayData` y `displayTotal`.
   - Al seleccionar editar o ver, el padre vuelve a usar el servicio con `getCharacter(id)` para obtener detalles completos del personaje.
   - Con esos detalles, el padre abre el modal hijo (`CharacterForm` o `CharacterDetailComponent`) pasando objetos de datos a `DynamicDialogConfig.data`.
   - El componente hijo de modal recibe los datos mediante `DynamicDialogConfig`, los presenta y puede devolver resultados con `ref.close(finalObject)`.
   - El padre puede suscribirse a `onClose` de la referencia del modal para recibir el objeto actualizado y eventualmente actualizar su estado local.

## 8. **Uso de Signals y estado reactivo:**
   - **Signals**: `characters`, `totalCount`, `isLoading` para estado reactivo.
   - **Inyección**: Usa `inject()` para `CharacterService` y `DialogService`.
   - **Reactividad**: Los signals actualizan la UI automáticamente al cambiar.

## 9. **Estilos y diseño:**
   - **SCSS**: Similar a otros componentes, con variables CSS para fondos, colores temáticos.
   - **Layout**: Usa `p-card` para contenedor, header con imagen de Goku.
   - **Tema**: Consistente con Dragon Ball Z (colores, fuentes Oswald).

## 10. **Flujo de aplicación:**
   1. Usuario navega a `/dashboard/characters`.
   2. Componente carga personajes iniciales (página 1, límite 10).
   3. CharacterTable muestra tabla con paginación y búsqueda.
   4. Usuario puede paginar, buscar, o seleccionar personaje.
   5. Al seleccionar: Se abre modal de detalles o edición según el tipo de acción.
   6. Modales permiten ver/editar sin salir de la vista.

## 11. **Conexiones con otros componentes:**
   - **Padre**: DashboardLayoutComponent (lo carga vía rutas).
   - **Hijos**: CharacterTable (maneja la tabla), HeadersComponent.
   - **Modales**: CharacterDetailComponent (detalles), CharacterForm (edición).
   - **Servicios**: CharacterService para datos.

Este componente es el núcleo de la gestión de personajes, proporcionando una interfaz completa para listar, buscar y gestionar personajes con integración de modales para acciones detalladas.
