Análisis completo del componente CharacterForm:

## 1. **Archivos del componente:**
   - **character-form.ts**: Componente principal en TypeScript.
   - **character-form.html**: Plantilla HTML.
   - **character-form.scss**: Estilos CSS/SCSS.
   - **character-form.spec.ts**: Archivo de pruebas unitarias.

## 2. **Funcionalidad general:**
   El componente CharacterForm es un formulario modal para editar o crear personajes de Dragon Ball. Permite modificar datos básicos (nombre, raza, género, afiliación), métricas de poder (Ki base y máximo), descripción, imagen, planeta de origen, transformaciones y fecha de registro. Incluye validaciones, preview de imagen y gestión dinámica de transformaciones.

## 3. **Dónde es invocado:**
   - **Modal dinámico**: Abierto desde `CharacterListComponent` en un modal usando `DialogService.open(CharacterForm, {...})`.
   - **Condición**: En `handleCharacterUpdate()` de CharacterList, cuando `type` es `true` (modo edición).
   - **Datos pasados**: Recibe `response` (datos del personaje) y opcionalmente `transformations` (transformación seleccionada).
   - **Header**: "Editar a {nombre del personaje}".
   - **Ancho**: 90% del viewport.

## 4. **Uso de servicios:**
   - **MessageService**: Para mostrar toasts de éxito/error al guardar o validar.
   - No utiliza servicios HTTP directamente; los datos se pasan vía config del modal.

## 5. **Procesamiento y manejo de datos:**
   - **Inicialización**: En `ngOnInit()`, llama a `initForm()` con datos del config.
   - **Parseo de Ki**: Función `parseKi()` limpia strings y convierte a números.
   - **Formulario reactivo**: Usa `FormBuilder` para crear `FormGroup` con validadores.
     - Campos: id, name, ki, maxKi, race, gender, description, image, affiliation, date, originPlanet (subgrupo), transformations (FormArray).
   - **Transformaciones**: `FormArray` dinámico; métodos `addTransformation()` y `removeTransformation()`.
   - **Imagen**: `onImageSelect()` convierte archivo a base64 y actualiza preview.

## 6. **Interfaz de usuario:**
   - **Secciones**:
     1. Imagen (con upload) y datos básicos (nombre, raza, género, afiliación).
     2. Planeta de origen (nombre, destruido, descripción).
     3. Métricas de Ki (base y máximo con InputNumber).
     4. Descripción (textarea).
     5. Transformaciones (lista dinámica con add/remove).
     6. Fecha de registro (DatePicker).
   - **Botones**: "GUARDAR CAMBIOS" (submit) y "CANCELAR".
   - **Preview de imagen**: Muestra imagen actual o placeholder.
   - **Estados**: Usa signals para `imagePreview`, `razas`, `facciones`.

## 7. **Validaciones y guardado:**
   - **Validadores**: Required en name, ki, gender, date; min(0) en ki/maxKi.
   - **Guardar**: `guardarCambios()` valida formulario, muestra toast de éxito/error, cierra modal con datos.
   - **Cancelar**: `onCancel()` cierra modal sin guardar.

## 8. **Uso de Signals y estado reactivo:**
   - **Signals**: `imagePreview` (preview de imagen), `razas` (opciones de raza), `facciones` (opciones de afiliación).
   - **Computed**: `planetaEstado` (calcula si planeta está destruido basado en checkbox).
   - **FormArray**: `transformationsArray` para acceso al array de transformaciones.

## 9. **Estilos y diseño:**
   - **SCSS**: Tema oscuro con gradientes naranjas, fondos traslúcidos, bordes redondeados.
   - **Personalización PrimeNG**: Overrides para inputs, selects, botones con colores temáticos.
   - **Layout**: Grid responsive, secciones organizadas.
   - **Tema**: Consistente con Dragon Ball Z (colores #fb8c00, fondos oscuros).

## 10. **Flujo de aplicación:**
   1. Usuario selecciona "editar" en CharacterList.
   2. CharacterList abre modal con CharacterForm, pasando datos.
   3. CharacterForm inicializa formulario con datos precargados.
   4. Usuario edita campos, añade/remueve transformaciones, sube imagen.
   5. Al guardar: Valida, muestra toast, cierra modal con datos actualizados.
   6. CharacterList recibe datos y podría procesarlos (aunque en código actual solo loguea).

## 11. **Conexiones con otros componentes:**
   - **Padre/Modal**: Invocado por CharacterListComponent.
   - **Hermanos**: Comparte lógica con CharacterDetailComponent (ambos muestran datos de personaje).
   - **Servicios**: MessageService para notificaciones.

Este componente proporciona una interfaz completa para edición de personajes, con gestión avanzada de datos y UI intuitiva.

Diagrama de flujo:
CharacterListComponent → DialogService.open(CharacterForm, data) → DynamicDialogConfig.data → CharacterForm.initForm(data) → Usuario edita formulario → guardarCambios() → MessageService.add() → ref.close(finalObject)
