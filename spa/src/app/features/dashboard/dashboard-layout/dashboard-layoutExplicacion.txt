Análisis completo del componente DashboardLayoutComponent:

## 1. **Archivos del componente:**
   - **dashboard-layout.component.ts**: Componente principal en TypeScript.
   - **dashboard-layout.component.html**: Plantilla HTML.
   - **dashboard-layout.component.scss**: Estilos CSS/SCSS.
   - **dashboard-layout.component.spec.ts**: Archivo de pruebas unitarias.

## 2. **Funcionalidad general:**
   El componente DashboardLayoutComponent es el layout principal del dashboard de la aplicación. Proporciona una estructura de navegación consistente con un menú superior, opciones de configuración (modo oscuro) y un área de contenido donde se renderizan las vistas hijas (personajes, estadísticas). También incluye funcionalidades de logout y una opción de prueba para el interceptor HTTP.

## 3. **Dónde es invocado:**
   - **Rutas**: Definido en `app.routes.ts` como el componente padre para la ruta `/dashboard`.
   - **Jerarquía**: Es el contenedor que carga las rutas hijas `/dashboard/characters` y `/dashboard/stats` a través de `<router-outlet>`.
   - **Protección**: La ruta `/dashboard` está protegida por `authGuard`, por lo que solo usuarios autenticados pueden acceder.
   - **Acceso**: Usuario llega aquí después del login exitoso, siendo la vista principal post-autenticación.

## 4. **Uso de servicios:**
   - **Router**: Para navegación programática (logout redirige a `/login`).
   - **CharacterService**: Solo para la función de prueba del interceptor (`getPrueba()`).
   - No utiliza servicios HTTP directamente en su lógica principal; delega a los componentes hijos.

## 5. **Interfaz de usuario y componentes hijos:**
   - **p-menubar**: Menú superior con navegación.
     - Logo de la aplicación.
     - Items del menú: "Personajes" (routerLink: 'characters'), "Estadísticas" (routerLink: 'stats'), "Prueba Interceptor" (command para probar error).
   - **app-dark-mode**: Componente compartido para alternar entre modo claro y oscuro.
   - **p-button**: Botón de "Salir" para logout.
   - **router-outlet**: Punto de montaje para las rutas hijas (CharacterListComponent, Stats).
   - **Fondo**: Imagen de fondo con patrón de puntos grises.

## 6. **Manejo de eventos y navegación:**
   - **Menú de navegación**: Los items con `routerLink` usan Angular Router para cambiar entre vistas hijas sin recargar la página.
   - **Prueba Interceptor**: Botón que llama a `pruebaInterceptor()`, el cual invoca `charService.getPrueba()` para simular un error HTTP y probar el interceptor global.
   - **Logout**: Método `logout()` que borra el token de localStorage y navega a `/login`, efectivamente cerrando la sesión.

## 7. **Uso de Signals y estado reactivo:**
   - **Signals**: `items` para la configuración del menú (array de MenuItem).
   - **Inyección**: Usa `inject()` para `Router` y `CharacterService`.
   - **Reactividad**: El menú se actualiza automáticamente si el signal `items` cambia, aunque en este caso es estático.

## 8. **Estilos y diseño:**
   - **SCSS**: Diseño responsivo con flexbox, variables CSS para colores temáticos (modo oscuro/claro).
   - **Layout**: Barra superior fija, contenido principal con padding y max-width.
   - **Tema**: Consistente con Dragon Ball Z (colores, logos, fondos).
   - **Media queries**: Adaptación para móviles (oculta toolbar en pantallas pequeñas).

## 9. **Flujo de aplicación:**
   1. Usuario autenticado accede a `/dashboard`.
   2. DashboardLayoutComponent carga con menú superior.
   3. Por defecto, redirige a `/dashboard/characters` (CharacterListComponent).
   4. Usuario puede navegar entre "Personajes" y "Estadísticas" usando el menú.
   5. Opciones adicionales: cambiar modo oscuro, probar interceptor, o salir (logout).
   6. El contenido de las vistas hijas se renderiza en `<router-outlet>`.

## 10. **Conexiones con otros componentes:**
   - **Padre**: Ninguno directo; es el punto de entrada del dashboard.
   - **Hijos**: Carga dinámicamente CharacterListComponent y Stats vía rutas.
   - **Compartidos**: DarkModeComponent para configuración global.
   - **Servicios**: CharacterService para pruebas; AuthService implícitamente (protección de ruta).

## 11. **Observaciones relevantes:**
   - Actúa como shell del dashboard, separando la navegación de la lógica de negocio.
   - La opción "Prueba Interceptor" es una herramienta de desarrollo para verificar el manejo de errores HTTP.
   - El diseño es modular y reutilizable, permitiendo agregar más opciones al menú fácilmente.

Este componente es el esqueleto del dashboard, proporcionando navegación y configuración mientras delega el contenido específico a sus rutas hijas.
