Análisis de las interfaces en `spa/src/app/shared/interfaces/models`:

## 1. **Interfaces disponibles:**
   - **character.model.ts**: Define interfaces para modelos de personajes, transformaciones y planetas.
   - **login-text.interface.ts**: Define interfaces para textos de login y modo oscuro.

## 2. **Análisis de character.model.ts:**
   - **Interfaces definidas**:
     - `Character`: Modelo básico de personaje (id, name, ki, etc.).
     - `OriginPlanet`: Datos del planeta de origen.
     - `Transformation`: Modelo de transformación (id, name, ki, image).
     - `Detail`: Modelo detallado de personaje con planeta y transformaciones.
     - `DetailEdit`: Versión editable de Detail, usada en formularios.
     - `ModalCharacter`: Para datos en modales.
   - **Dónde es invocado**:
     - **StatsComponent**: Importa `Transformation` y `Detail` para manejar datos del gráfico y modales.
     - **CharacterDetailComponent**: Importa `Transformation` y `Detail` para mostrar información en el modal.
     - **CharacterListComponent**: Importa `Detail` para tipar datos en modales.
     - **CharacterFormComponent**: Importa `Transformation`, `Detail`, `DetailEdit` para formularios y validaciones.
     - **CharacterTableComponent**: Importa `Character` para tipar el evento de selección.
   - **Por qué**: Proporciona tipado fuerte para datos de la API de Dragon Ball, asegurando consistencia en componentes que manejan personajes. Facilita desarrollo con IntelliSense y previene errores de tipos.

## 3. **Análisis de login-text.interface.ts:**
   - **Interfaces definidas**:
     - `LoginText`: Para textos del componente de login (title, subtitle, labels, button).
     - `DarkMode`: Para textos de modo oscuro (DARK, LIGHT).
   - **Dónde es invocado**:
     - Aunque las interfaces no se importan directamente, los datos se cargan desde `login-texts.json` en:
       - **LoginComponent**: Importa `loginData` para textos dinámicos.
       - **DarkModeComponent**: Importa `loginData` para textos de modo (aunque usa signal interno).
     - Las interfaces sirven para tipar el JSON si se usaran en TypeScript puro.
   - **Por qué**: Permite internacionalización y centralización de textos. El JSON se importa directamente para evitar complejidad, pero las interfaces definen la estructura esperada.

## 4. **Patrones comunes en las interfaces:**
   - **Tipado fuerte**: Usadas para type safety en componentes que manejan datos complejos.
   - **Reutilización**: Compartidas entre múltiples componentes (stats, character-list, forms, etc.).
   - **Separación**: Mantienen la lógica de UI separada de la definición de datos.
   - **Extensibilidad**: Fácil agregar campos sin cambiar código existente.

## 5. **Conclusión:**
   Estas interfaces son fundamentales para el tipado en la aplicación, siendo invocadas en componentes clave que manejan personajes y textos. `character.model.ts` es usado extensivamente en el dashboard para datos dinámicos, mientras `login-text.interface.ts` soporta textos estáticos. Mejoran la mantenibilidad y reducen errores en desarrollo.

Diagrama de flujo:
API response → CharacterService → data tipada con Character/Detail/Transformation → Componentes Stats / CharacterList / CharacterForm / CharacterDetail

Textos:
login-texts.json → LoginComponent / DarkModeComponent → UI dinámico
